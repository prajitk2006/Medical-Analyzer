Project 2 — Data Analytics
📊 Hospital Operations Analytics Dashboard
Why this impresses Siemens specifically: Siemens launched AI services that span scheduling, image acquisition to reporting — and they're building tools to help hospitals address staff shortages and organise shifts more flexibly. Imaging Technology News A dashboard that analyses exactly these operational problems shows you understand their product space.
What to build:
A Power BI + Python analytics project on hospital patient flow data — wait times, bed occupancy, department load, readmission rates. Use the NHS Hospital Episode Statistics open dataset or the Healthcare Analytics dataset on Kaggle.
Step by step:

Python (Pandas) pipeline to clean and transform raw hospital data — flag anomalies, handle missing values, document every transformation
Feature engineering — calculate average wait time by department, peak load hours, readmission rate by diagnosis group
Build a Power BI dashboard with 4 views:

Department throughput over time
Peak vs off-peak patient load heatmap
Readmission risk by patient age group
Bed occupancy vs staffing levels


Write a short analysis document (1–2 pages) with 3 concrete operational recommendations from your findings
All Python code in GitHub with tests for transformation functions

What makes it stand out for Siemens:

Frame the entire project around reducing clinician workload — not just "here's a dashboard." Siemens' own pitch is freeing up doctors from routine admin. Mirror that language.
Add a trend forecast — even a simple linear regression or moving average showing predicted bed demand next week. This nudges it from analytics into predictive analytics
In your README, explicitly connect findings to a real operational decision (e.g. "department X needs 2 additional staff on Tuesday mornings based on load data")

Stack: Python, Pandas, Power BI, Matplotlib/Seaborn, Git
Time to build: 2–3 weeks
