Project Datasets Dictionary

ETL Processed JSON

The dashboard is powered by a set of static JSON files generated through our Python ETL pipeline. These files represent the cleaned, transformed, and aggregated state of our raw hospital operations data.

Dataset Name Description Format Action
detailed_patients.json Comprehensive patient journey records including admission, discharge, diagnosis, length of stay, and readmission risk markers. Used for the Reports table. JSON
bed_occupancy_daily.json Historical daily aggregation of bed occupancy rates. Used for utilization metrics and trend charting. JSON
bed_forecast.json Predictive linear regression output mapping expected bed demand over the next 7 days based on seasonal and historical trends. JSON
peak_load_heatmap.json Aggregated patient arrival counts bucketed by hour of day and day of week to identify emergency department bottlenecks. JSON
dept_wait_metrics.json Time-series data tracking the average wait time from admission to first physician contact across different hospital departments. JSON
staff_details.json Directory of attending physicians and nurses, including current assignments and departmental mapping. JSON
staffing_needs_by_dept_hour.json Calculated staffing recommendations based on patient load volume and acuity levels across a 24-hour cycle. JSON
readmission_rates_by_age.json Aggregated percentages of patient 30-day readmissions segmented into primary age brackets. JSON
